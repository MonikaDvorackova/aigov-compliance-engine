## Phase 1 integrity prerequisites (digest manifest + artifact digest)

This report clarifies the **Phase 1 governance boundary** for production promotion gates.

### Governance prerequisites (Phase 1)

- **Digest manifest prerequisite**: every evidence pack MUST include `evidence_digest_manifest.json` with
  `events_content_sha256` (portable digest over evidence events) and MUST be consistent with the on-disk bundle
  `<run_id>.json` (recomputed digest equality).
- **Artifact digest prerequisite**: every `model_promoted` evidence event MUST include an **artifact digest**
  (`artifact_sha256` or `artifact_digest_sha256`) as the **source-of-truth** for “what was promoted”.
  This is not satisfied by the evidence events hash alone.

### Server-side vs CI-side enforcement

- **Server-side (ingest)**:
  - Reject `model_promoted` events that do not carry an artifact digest (fail-closed).
  - Continue enforcing existing tenant isolation via API key boundaries and `/ready` as authoritative readiness.
- **CI-side (production hosted gate)**:
  - Verify `<run_id>.json` recomputed portable digest equals `evidence_digest_manifest.json.events_content_sha256`.
  - Verify that a `model_promoted` artifact digest exists in evidence (even if the artifact file is unavailable).
  - Optionally (when artifact file is available), verify artifact file SHA256 equals the promoted digest.

## Evaluation gate

The evaluation gate asserts that the run is **cryptographically self-consistent** (bundle ↔ digest manifest)
and that the promoted artifact is **artifact-bound** (stable digest present; optional file verification).

## Human approval gate

Human approval is evaluated as an explicit, auditable evidence event (`human_approved` scoped to `model_promoted`)
and remains distinct from the digest/traceability prerequisites above.
